# Day 01 — The Problem MCP Solves

## Before MCP — AI Was Isolated

```
You (human) ──▶ AI (Claude)

You: "What's in my database?"
AI:  "I don't know. I can't access your database."

You: "Read my files."
AI:  "I can't. I have no access."
```

AI could only work with what you **copy-paste** into the chat. It had no way to reach your tools, files, databases, or services.

---

## After MCP — AI Connects to Everything

```
You ──▶ AI ──▶ MCP Server ──▶ Your Database
                           ──▶ Your Files
                           ──▶ GitHub
                           ──▶ Slack
                           ──▶ Anything
```

**MCP is a bridge** between AI and the outside world.

---

## Real-World Analogy

Think of your **phone**:
- Phone alone = limited (can only call)
- Phone + apps = powerful (maps, banking, social media, etc.)

**MCP servers are like apps for AI.** Each one gives AI a new ability.

---

## Why Not Just Use APIs Directly?

| Approach | Problem |
|----------|---------|
| Every AI tool builds its own integration | Fragmented, no standard |
| Copy-paste data into chat | Slow, error-prone, limited |
| Custom plugins per platform | Lock-in, not portable |
| **MCP (standard protocol)** | **One standard, works everywhere** |

---

## Key Takeaway

> MCP = **Universal plug system for AI.** Build a server once, it works with any MCP-compatible AI client. Just like USB-C works with any device.
